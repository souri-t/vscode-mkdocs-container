# 運用設計

## 環境設定

### 設定ファイル

- `app.properties`: API ベース URL、タイムアウト、ログ設定
- `firebase-config.json`: Push 通知設定
- `secure-preferences`: 認証トークンやユーザー設定を保持

### 環境別設定

| 環境 | データベース | 外部API |
|------|-------------|---------|
| 開発 | Room ローカル DB | `https://dev-api.todo.example.com/v1` |
| ステージング | Room ローカル DB | `https://stg-api.todo.example.com/v1` |
| 本番 | Room ローカル DB | `https://api.todo.example.com/v1` |

### 切り替え方針

ビルドタイプ `debug` `staging` `release` により、接続先 API とログレベルを切り替える。

## ログ設計

### ログ種別

- **操作ログ**: ログイン、タスク登録、更新、削除
- **同期ログ**: API 同期開始、完了、失敗
- **エラーログ**: アプリ例外、API 通信失敗、通知登録失敗

### ログ出力項目

| 項目 | 内容 |
|------|------|
| タイムスタンプ | ISO 8601 |
| ログレベル | DEBUG / INFO / WARN / ERROR |
| ユーザーID | ログイン済みユーザー ID |
| 端末ID | アプリ初回起動時に発行される識別子 |
| 操作内容 | 実行した画面操作や同期イベント |
| メッセージ | 詳細メッセージまたはエラー内容 |

### 保管場所・ローテーション

- 端末内ログはアプリ専用領域へ保存する
- 直近 7 日分または 10MB を上限としてローテーションする
- 障害調査時のみユーザー同意のうえで収集する

## 同期運用

- アプリ起動時と手動更新時に差分同期を行う
- 更新競合時はサーバー側の内容を再取得し、端末側へ通知する
- 3 回連続で同期失敗した場合はエラーバナーを表示する
